# Use Case 2: Вход пользователя в систему

| Поле | Описание |
|------|----------|
| **ID** | UC-02 |
| **Название** | Вход пользователя в систему |
| **Акторы** | Пользователь (неавторизованный, но имеющий учётную запись), Система |
| **Предусловия** | Пользователь не авторизован; имеет доступ к интернету; есть зарегистрированная учётная запись (email и пароль) |
| **Постусловия (успех)** | Пользователь успешно авторизован; перенаправляется на главную страницу; становится доступным функционал, требующий авторизации |
| **Постусловия (неудача)** | Введённые данные не сохранены; пользователь остаётся неавторизованным; сессия не создаётся |

## Основной поток

1. Пользователь переходит на страницу входа (нажимает «Войти» в шапке сайта).
2. Система отображает форму входа с полями:
   - Email (обязательное)
   - Пароль (обязательное)
   - Запомнить меня (чекбокс)
3. Пользователь вводит свой email и пароль и нажимает кнопку «Войти».
4. Система проверяет наличие учётной записи с указанным email.
5. Если запись найдена, система сравнивает введённый пароль с хешем, хранящимся в базе данных.
6. При успешном совпадении система создаёт сессию для пользователя.
7. Система перенаправляет пользователя на главную страницу или в личный кабинет (или на ту страницу, с которой он пришёл, если была запрошена защищённая страница).
8. Пользователь получает доступ к функционалу.

## Альтернативные потоки

**Неверный email**  
Точка входа: Шаг 4  
4.1 Система не находит учётную запись с указанным email.  
4.2 Сообщение об ошибке: «Неверный email или пароль».  
4.3 Возврат к шагу 3.

**Неверный пароль**  
Точка входа: Шаг 5  
5.1 Система находит учётную запись, но введённый пароль не совпадает с хешем.  
5.2 Сообщение об ошибке: «Неверный email или пароль».  
5.3 Возврат к шагу 3.

**Аккаунт заблокирован / не активирован**  
Точка входа: Шаг 5  
5.1 Система находит учётную запись, пароль верен, но статус аккаунта — «заблокирован» или «не подтверждён» (после регистрации требуется подтверждение email).  
5.2 Сообщение: «Ваш аккаунт заблокирован» или «Пожалуйста, подтвердите email, перейдя по ссылке в письме».  
5.3 Use Case завершается.

**Слишком много попыток входа**  
Точка входа: Шаг 4  
4.1 Система обнаруживает, что с данного IP или для данного email превышено количество неудачных попыток входа за короткий промежуток времени.  
4.2 Система временно блокирует возможность входа (на 15 минут) и выводит сообщение: «Слишком много попыток входа. Попробуйте позже».  
4.3 Use Case завершается.

## Исключительный поток

**Ошибка при проверке учётной записи**  
Точка входа: Шаг 4  
4.1 При запросе к базе данных происходит ошибка.  
4.2 Сообщение: «Временно недоступно. Попробуйте позже».  
4.3 Use Case завершается.
