# Use Case 5: Добавление нового квеста в партию

| Поле | Описание |
|------|----------|
| **ID** | UC-05 |
| **Название** | Добавление нового квеста в партию |
| **Акторы** | Мастер (участник партии с ролью «мастер»), Система |
| **Предусловия** | Мастер авторизован и находится в управлении своей партии; партия уже создана |
| **Постусловия (успех)** | Квест сохранён и отображается в общем списке квестов партии; виден всем участникам (игрокам и мастеру); мастер может позже редактировать или менять статус квеста |
| **Постусловия (неудача)** | Квест не создан; данные не сохранены; форма остаётся заполненной с подсветкой ошибок |

## Основной поток

1. Мастер переходит в раздел «Квесты» партии и нажимает кнопку «Создать квест».
2. Система отображает форму создания квеста с полями:
   - Название квеста (обязательное)
   - Описание (текст, поддерживающий форматирование)
   - Статус (активен, выполнен, провален) – по умолчанию «активен»
   - Связанные NPC (можно выбрать из списка существующих NPC партии или оставить пустым)
   - Уровень сложности (необязательно)
3. Мастер заполняет поля и нажимает «Сохранить».
4. Система проверяет корректность введённых данных (название не пустое).
5. Система сохраняет квест в базе данных, привязывая его к партии.
6. Система отображает созданный квест в общем списке.

## Альтернативные потоки

**Создание квеста без NPC**  
Точка входа: Шаг 2  
2.1 Мастер оставляет поле «Связанные NPC» пустым.  
2.2 Далее поток продолжается с шага 3.

**Название квеста уже существует**  
Точка входа: Шаг 4  
4.1 Система обнаруживает, что квест с таким названием уже есть в этой партии.  
4.2 Сообщение: «Квест с таким названием уже существует. Пожалуйста, выберите другое название».  
4.3 Возврат к шагу 3 для исправления.

## Исключительный поток

**Ошибка сохранения**  
Точка входа: Шаг 5  
5.1 При сохранении произошёл сбой.  
5.2 Сообщение: «Ошибка при создании квеста. Попробуйте снова».  
5.3 Use Case завершается.

## Бизнес-правила

- **BR-05-01:** Название квеста должно быть уникальным в пределах партии.
- **BR-05-02:** Название квеста должно содержать от 3 до 60 символов.
- **BR-05-03:** Квест может иметь только один из статусов: «активен», «выполнен», «провален».
- **BR-05-04:** В одной партии может быть не более 50 активных квестов одновременно.
