# Use Case 4: Создание нового персонажа в партии

| Поле | Описание |
|------|----------|
| **ID** | UC-04 |
| **Название** | Создание нового персонажа в партии |
| **Акторы** | Игрок (зарегистрированный пользователь, участник партии), Система |
| **Предусловия** | Пользователь авторизован; является участником хотя бы одной партии; в партии у игрока ещё нет персонажа |
| **Постусловия (успех)** | Персонаж сохранён и доступен для просмотра/редактирования игроком; мастер видит персонажа в общем списке; производные параметры (модификаторы, AC, хиты) рассчитаны автоматически |
| **Постусловия (неудача)** | Персонаж не создан; данные не сохранены; форма остаётся заполненной с подсветкой ошибок |

## Основной поток

1. Игрок заходит в партию, переходит в раздел «Мои персонажи» и нажимает «Создать персонажа».
2. Система отображает форму создания персонажа, адаптированную под игровую систему партии (для D&D 5e поля: имя, раса, класс, уровень, характеристики).
3. Игрок заполняет поля:
   - Имя персонажа (обязательное)
   - Раса (выбор из списка)
   - Класс (выбор из списка)
   - Уровень (число, по умолчанию 1)
   - Характеристики (Сила, Ловкость, Телосложение, Интеллект, Мудрость, Харизма)
4. По мере заполнения система в реальном времени рассчитывает и отображает производные параметры:
   - Модификаторы характеристик
   - Класс брони (AC)
   - Хиты (HP)
   - Модификаторы навыков
5. Игрок нажимает «Сохранить персонажа».
6. Система проверяет корректность данных (обязательные поля заполнены, значения в допустимых диапазонах).
7. Система сохраняет персонажа, связывая его с игроком и партией.
8. Система отображает полный лист созданного персонажа.

## Альтернативные потоки

**Ручной режим редактирования**  
Точка входа: Шаг 4  
4.1 Игрок включает чекбокс «Ручной режим».  
4.2 Система отключает автоматические расчёты и позволяет вручную вводить любые значения производных параметров.  
4.3 Далее поток продолжается с шага 5.

**Обязательное поле не заполнено**  
Точка входа: Шаг 6  
6.1 Система обнаруживает, что поле «Имя персонажа» пустое.  
6.2 Система подсвечивает поле красным и выводит сообщение: «Имя персонажа обязательно».  
6.3 Возврат к шагу 3 для исправления.

## Исключительный поток

**Ошибка сохранения**  
Точка входа: Шаг 7  
7.1 При сохранении произошла ошибка.  
7.2 Сообщение: «Не удалось сохранить персонажа. Попробуйте позже».  
7.3 Use Case завершается.

## Бизнес-правила

- **BR-04-01:** В рамках одной партии игрок может иметь только одного активного персонажа.
- **BR-04-02:** Имя персонажа должно быть уникальным в пределах партии.
- **BR-04-03:** Уровень персонажа должен быть целым числом от 1 до 20.
- **BR-04-04:** Базовые характеристики должны быть в диапазоне от 1 до 30.
- **BR-04-05:** Модификатор характеристики рассчитывается по формуле: `(значение - 10) / 2` (округление вниз).
