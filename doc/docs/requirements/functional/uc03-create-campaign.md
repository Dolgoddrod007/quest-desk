# Use Case 3: Создание новой игровой партии

| Поле | Описание |
|------|----------|
| **ID** | UC-03 |
| **Название** | Создание новой игровой партии |
| **Акторы** | Мастер (зарегистрированный пользователь с ролью «мастер»), Система |
| **Предусловия** | Пользователь авторизован; имеет статус «мастер» |
| **Постусловия (успех)** | В списке партий мастера появляется новая запись; сгенерирована уникальная пригласительная ссылка; мастер может перейти к управлению партией |
| **Постусловия (неудача)** | Партия не создана; введённые данные не сохранены (форма остаётся с подсветкой ошибок) |

## Основной поток

1. Мастер переходит в раздел «Мои партии» и нажимает кнопку «Создать новую партию».
2. Система отображает форму создания партии с полями:
   - Название партии (обязательное)
   - Описание (необязательное)
   - Игровая система (D&D 5e)
3. Мастер заполняет обязательные поля и нажимает «Создать».
4. Система проверяет уникальность названия в пределах аккаунта мастера.
5. Система сохраняет новую партию в базе данных и автоматически добавляет создателя с ролью «Мастер».
6. Система генерирует уникальную пригласительную ссылку и отображает её мастеру.
7. Мастер копирует ссылку для последующей рассылки игрокам.

## Альтернативный поток

**Название партии уже существует**  
Точка входа: Шаг 4  
4.1 Система обнаруживает, что у мастера уже есть партия с таким названием.  
4.2 Сообщение: «Партия с таким названием уже существует. Пожалуйста, выберите другое название».  
4.3 Возврат к шагу 2 для исправления.

## Исключительный поток

**Ошибка сохранения в БД**  
Точка входа: Шаг 5  
5.1 При сохранении произошёл сбой.  
5.2 Сообщение: «Не удалось создать партию. Попробуйте позже».  
5.3 Use Case завершается.

## Бизнес-правила

- **BR-03-01:** Название партии должно быть уникальным в пределах аккаунта мастера.
- **BR-03-02:** Название партии должно содержать от 3 до 50 символов.
- **BR-03-03:** Пригласительная ссылка генерируется как случайная строка и действует 7 дней.
