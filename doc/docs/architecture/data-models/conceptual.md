# Концептуальная модель данных

*Ссылка на доску Unidraw: [открыть](https://unidraw.io/app/board/fb38eb3c4e3ca5ced360?allow_guest=true)*



## Основные сущности

- **Пользователь (User)** — содержит учётные данные и профиль.
- **Кампания (Campaign)** — игровая партия, создаваемая мастером.
- **Участник кампании (CampaignMember)** — связь пользователя с кампанией и ролью.
- **Персонаж (Character)** — лист персонажа внутри кампании.
- **Квест (Quest)** — задание мастера.
- **NPC (NpcTemplate)** — шаблон неигрового персонажа.
- **Сессия (Session)** — запланированная игровая встреча.
- **Доступность сессии (SessionAvailability)** — отметка участника о возможности прийти.
- **Журнал сессии (SessionLog)** и **Комментарии (LogComment)**.
- **Лог событий (EventLog)** — аудит ключевых действий.
- **Ежедневная статистика (UserStatsDaily)** — агрегация для аналитики.

Все связи подобраны так, чтобы обеспечить требования MVP.
