# Риски и управление ими

| Риск | Вероятность | Влияние | Стратегия |
|------|-------------|---------|-----------|
| Сложность реализации авторасчётов | Средняя | Высокое | Использовать готовые библиотеки, упростить логику до базовых формул D&D 5e. |
| Недостаточная гибкость для разных систем | Средняя | Среднее | Заложить расширяемую архитектуру, в первой версии поддерживать только D&D 5e. |
| Низкое тестирование без реальных пользователей | Высокая | Среднее | Привлечь знакомых игроков, создать фокус-группу. |
| Технические проблемы с синхронизацией данных | Низкая | Высокое | Надёжная СУБД, блокировки, версионирование. |
| Срыв сроков | Средняя | Высокое | Итеративная разработка, чёткие приоритеты MVP. |
| Изменение требований в процессе | Средняя | Среднее | Журнал требований, согласование изменений. |
| Неудобный интерфейс на мобильных | Средняя | Среднее | Адаптивная вёрстка с самого начала. |
| Отсутствие интереса к продукту | Средняя | Среднее | Опросы в сообществах, подтверждение гипотезы. |
| Юридические риски (товарные знаки) | Низкая | Низкое | Не использовать охраняемые знаки. |
